언어 번역 방법

    
1.컴파일러 언어: 모든 소스를 한꺼번에 번역한다. 번역하다가 오류가 발생하면 오류가 없을 때까지 수정해야 다시 번역해서 실행한다.
                     ex) java, C++, C  등
                     장점: 에러메세지가 아주 다양해서 에러메세지를 보면 어디가 오류인지 바로 알 수 있다
                     단점: 배우기 어렵다


    
2.인터프리터 언어:> 한번에 한줄씩 번역하여 에러가 없으면 바로 실행한다
                     ex) javascript, node.js, Python  등
                      보통은 교육용 언어에만 적용되었는데 현재는 실용적인 언어들에도 많이 적용된다
                      장점: 보통 언어가 배우기 쉽다
                      단점: 에러 메세지가 뭘 말하는지 알기 어려우며 시간이 지나도 디버깅 속도가 별로 나아지지 않는다


    
3.스크립트: 번역기가 별도로 존재하지 않는다 보통 다른 프로그램에 포함되어서 돌아가는 언어들에 붙이는 총칭이다. 
                Javascript는 java라는 언어의 형식을 따랐을 뿐 자바하고 아무 관계가 없는 언어이고 자바스크립트의 번역기는 브라우저다
                브라우저의 본래일은 서버에 요청을 하고, 응답을 받은 문서를 파싱하여 호면에 보여주는 일이 본업이고 Javascript번역은 부가적인 일이다

   
    
4.콘솔(console): 키보드와 모니터를 묶어서 부르는 말임
